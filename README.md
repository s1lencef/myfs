# Файловая система на блочном устройстве

Реализация в рамках дисциплины «Разработка модулей ядра Linux».
Целевое ядро: **6.12.90** (код использует API, актуальный для ветки 6.12).

Проект состоит из двух частей:

- `module/` — модуль ядра (сама файловая система + ioctl);
- `userspace/` — демонстрационная программа на C++ (обход диска, CLI для ioctl).

---

## 1. Что реализовано 

| Требование | Где реализовано |
|---|---|
| ФС на базе существующего блочного устройства | `myfs.c`, монтирование через `mount_bdev` |
| Две копии суперблока (начало + смещение N секторов) | `myfs_write_super_copies`, параметры `sb_offset` / `sb_copy_offset` |
| Проверка целостности суперблока (хеш-сумма) | `myfs_sb_checksum` (CRC32), проверка в `myfs_read_super_at` |
| Каждый сектор/участок = файл, все файлы созданы при инициализации | `myfs_format` создаёт заголовки всех файлов |
| Все файлы одинакового размера, от 1 до M секторов | параметр `max_file_sectors` (M) |
| Монтирование в `/mnt` | `mount -t myfs ... /mnt/myfs` |
| Регистрация в VFS, файлы видны в userspace | `register_filesystem`, `myfs_iterate`, `myfs_lookup` |
| Чтение/запись (остальные операции не нужны) | `myfs_read`, `myfs_write` |
| Аргумент модуля: имя диска | `device_name` |
| Аргумент модуля: смещения N для двух копий | `sb_offset`, `sb_copy_offset` |
| Аргумент модуля: макс. длина имени | `max_name_len` |
| Аргумент модуля: макс. размер файла в секторах (M) | `max_file_sectors` |
| IOCTL: обнуление всех файлов | `MYFS_IOC_ZERO_ALL` |
| IOCTL: стереть ФС | `MYFS_IOC_ERASE_FS` |
| IOCTL: список метаинформации (хеши) | `MYFS_IOC_GET_META` |
| IOCTL: маппинг секторов файла | `MYFS_IOC_GET_MAP` |
| Userspace: запись+чтение случайного числа в каждый файл | `myfs_tool ... test` |
| Userspace: CLI для ioctl | `myfs_tool ... {zero,erase,meta,map N}` |

### mfs_tool.cpp

Userspace-утилита для тестирования и управления файловой системой MYFS через `ioctl`.

#### Возможности

- Проверка чтения/записи файлов (`test`)
- Обнуление всех файлов (`zero`)
- Стирание файловой системы (`erase`)
- Получение метаданных файлов (`meta`)
- Просмотр маппинга секторов файла (`map`)

### myfs.c

MYFS — учебная файловая система для Linux, работающая поверх блочного устройства.

ФС реализована как kernel module и поддерживает:
- монтирование через VFS
- чтение и запись файлов
- двойной суперблок с CRC32
- ioctl-интерфейс
- отображение файлов как обычных `fileN`

#### Возможности

- Две копии суперблока:
  - primary
  - backup
- Проверка целостности суперблока через CRC32
- Автоматическое форматирование устройства при отсутствии валидной ФС
- Фиксированный размер файлов
- Один файл = непрерывный диапазон секторов
- Поддержка:
  - `read`
  - `write`
  - `ioctl`
  - `readdir`
  - `lookup`

#### Архитектура

###### Структура диска

```text id="w4pn4n"
+-------------------+
| Primary SB        |
+-------------------+
| ...               |
+-------------------+
| Backup SB         |
+-------------------+
| file0             |
+-------------------+
| file1             |
+-------------------+
| file2             |
+-------------------+
```
### setup.sh

Скрипт автоматической подготовки окружения для MYFS.

Скрипт выполняет полный цикл запуска файловой системы:

1. Создаёт файл-образ диска
2. Подключает его как loop-устройство
3. Загружает kernel module `myfs.ko`
4. Монтирует файловую систему MYFS

После выполнения ФС готова к работе и доступна в `/mnt/myfs`.

### teardown.sh

Скрипт очистки демонстрационного окружения MYFS.

Скрипт корректно завершает работу файловой системы:

1. Размонтирует MYFS
2. Выгружает kernel module `myfs`
3. Отвязывает loop-устройство

## 2. Формат данных на диске

Размер «сектора» в ФС зафиксирован — 512 байт.

```
сектор sb_offset (обычно 0)   -> первая копия суперблока
сектор sb_copy_offset          -> вторая (резервная) копия суперблока
first_data_sector .. конец     -> файлы, по M секторов на файл
```

`first_data_sector = max(sb_offset, sb_copy_offset) + 1` — данные начинаются
после обеих копий суперблока.

Суперблок (`struct myfs_super_block`) хранит магию, версию, геометрию
(размер устройства, смещение копии, M, число файлов, первый сектор данных)
и контрольную сумму. Контрольная сумма считается по всей структуре с
обнулённым полем `checksum`.

Каждый файл начинается с заголовка (`struct myfs_file_header`): магия, длина
полезных данных и их CRC32. Полезные данные идут сразу после заголовка.

## 4. Запуск и демонстрация

Удобнее всего использовать готовые скрипты (требуют root).

### 4.1. Автоматическая настройка

```bash
cd module
sudo IMG_SIZE_MB=32 SB_COPY_OFFSET=64 M=2 ./setup.sh
```

Скрипт создаст образ `/tmp/myfs.img`, привяжет его к loop-устройству,
загрузит модуль с параметрами и смонтирует ФС в `/mnt/myfs`.
В конце он выведет имя использованного loop-устройства.

### 4.2. Демонстрация записи/чтения по всем файлам

```bash
cd userspace
./myfs_tool /mnt/myfs test
```

Программа в каждый файл пишет случайное 64-битное число и читает обратно,
сверяя значения. В конце печатает сводку «N OK, M FAILED».

### 4.3. Вызов IOCTL через CLI

```bash
# Список метаинформации (хеши) по всем файлам:
./myfs_tool /mnt/myfs meta

# Маппинг секторов для файла с индексом 0:
./myfs_tool /mnt/myfs map 0

# Обнулить все файлы:
./myfs_tool /mnt/myfs zero

# Стереть ФС (затирает оба суперблока):
./myfs_tool /mnt/myfs erase
```

После `erase` файловую систему нужно размонтировать; повторное монтирование
того же устройства приведёт к переинициализации (форматированию).

### 4.4. Снос окружения

```bash
cd module
sudo LOOP_DEV=/dev/loopX ./teardown.sh
# либо просто (скрипт сам найдёт loop по образу):
sudo ./teardown.sh
```

## 5. Ручной запуск (без скриптов)

```bash
# 1. Создать образ и loop-устройство
dd if=/dev/zero of=/tmp/myfs.img bs=1M count=32
sudo losetup --find --show /tmp/myfs.img      # -> /dev/loop0 (например)

# 2. Загрузить модуль с параметрами
sudo insmod module/myfs.ko \
    device_name=/dev/loop0 \
    sb_offset=0 \
    sb_copy_offset=64 \
    max_name_len=32 \
    max_file_sectors=2

# 3. Смонтировать
sudo mkdir -p /mnt/myfs
sudo mount -t myfs /dev/loop0 /mnt/myfs

# 4. Посмотреть файлы
ls -la /mnt/myfs

# 5. Размонтировать и выгрузить
sudo umount /mnt/myfs
sudo rmmod myfs
sudo losetup -d /dev/loop0
```

Логи модуля смотреть через `dmesg | tail`.
